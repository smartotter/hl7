IHE/MESA Message Sequences

The IHE/MESA tests are based on message sequences that exercise features of the IHE Technical Framework.  We have defined different sequences of messages and labeled them 101, 102, etc.  In most of the cases, we expect the actors under test to follow the sequences exactly as documented.  For example, it is common to think of a registration message (ADT), request for procedure (ORM) and a scheduling message (ORM) in that order.

In some cases, we document a message sequence, but the tests will allow slight deviations.  An example of this would be the sequence of MPPS messages, C-Store events and Storage Commitment requests produced by a modality.  The modality may perform some of these actions in a sequence that is different from the MESA documented sequence.  The MESA tests should allow for reasonable deviations.

The documents/links below are provided to show the entire message sequence involving all actors.  The tests for your actor may include a subset of the messages documented below.  Corrections for typos are welcome.  If we have contrived a message sequence that does not make sense clinically, we would especially like to hear your comments.
